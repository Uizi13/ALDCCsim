ALDCCsim is a C++ command-\/line tool designed to solve linear electrical circuits using Modified Nodal Analysis (MNA). The solver supports resistors, independent voltage sources, and independent current sources. It also features a bisection-\/based optimization engine to find specific component values required to achieve a desired circuit output (voltage, current, or power).\hypertarget{details-page_autotoc_md22}{}\doxysubsection{\texorpdfstring{Below is a full description of the program focusing on Analysis, Internal Specification and Testing. For External Specification read the {main} page}}\label{details-page_autotoc_md22}

\hypertarget{details-page_autotoc_md24}{}\doxysection{\texorpdfstring{Problem and analysis}{Problem and analysis}}\label{details-page_autotoc_md24}


Based on the provided code and the task description image, the core problem is the development of a software simulation tool capable of analyzing linear direct current (DC) circuits. The fundamental objective is to parse a textual representation of a circuit, commonly referred to as a netlist, and calculate the steady-\/state operating point for every component. This involves determining the voltage across, current through, and power dissipated by each resistor, voltage source, and current source in the network.

The mathematical foundation selected to solve this problem is Modified Nodal Analysis (MNA). This approach is necessary because the circuit elements include independent voltage sources, which standard nodal analysis cannot easily handle without supernodes. MNA addresses this by expanding the system of linear equations to solve simultaneously for node potentials and the currents flowing through voltage sources. The program can construct an augmented matrix where rows correspond to Kirchhoff\textquotesingle{}s Current Law (KCL) for each node and the constitutive equations for voltage sources, solving the system Ax=b using Gaussian elimination with partial pivoting to ensure numerical stability.

Several improvements can be made on top of what the task specifies. I decided to add a simple parallel voltage source check and a check for nodes with only current sources connected since these configurations result in the circuit being indeterminate. Obviously in some cases we need the output to be more precise than others and too high precision at all times results in the output being less readable. This is why I also added a third command line switch allowing the user to specify precision of output parameters.

The task mentions only solving a system with known element data and outputting currents, voltages and power balance, but in real use cases I found that I have to perform the opposite almost just as often. This is why I additionally implemented search mode which uses bisection to find some element parameter knowing one of the currents, voltages or power balances. The program still isn\textquotesingle{}t suitable for solving more elaborate problems, but this add-\/on makes its applicability significantly higher.

I also decided to replace the {\ttfamily I} current source symbol from the task decription with {\ttfamily J} as it is more commonly used and can\textquotesingle{}t be misinterpretted as current.\hypertarget{details-page_autotoc_md25}{}\doxysection{\texorpdfstring{Internal specification}{Internal specification}}\label{details-page_autotoc_md25}
\hypertarget{details-page_autotoc_md26}{}\doxysubsection{\texorpdfstring{Global Context}{Global Context}}\label{details-page_autotoc_md26}
The simulation state is aggregated into a central data structure, \doxylink{struct_context}{Context}, which serves as the primary data transfer object between modules.


\begin{DoxyItemize}
\item Element Storage:\+ All circuit components are stored in a std::\+vector\texorpdfstring{$<$}{<}\doxylink{struct_element}{Element}\texorpdfstring{$>$}{>}, where each Element contains its type, connection nodes, value, and computed results (voltage, current, power).
\item Equation System:\+ The MNA system is represented by an augmented matrix (std::\+vector$<$std::\+vector$<$double$>$$>$ equations), where the number of unknowns corresponds to the maximum node index plus the number of independent voltage sources.
\item Bisection State:\+ To support the optimization mode, the context holds a pointer to a variable\+\_\+elem (the component being tuned) and a \doxylink{struct_given_value}{Given\+Value} structure defining the target metric (desired\+\_\+value) and type (v\+\_\+type).
\end{DoxyItemize}\hypertarget{details-page_autotoc_md27}{}\doxysubsection{\texorpdfstring{Mathematical Core}{Mathematical Core}}\label{details-page_autotoc_md27}
\hypertarget{details-page_autotoc_md28}{}\doxysubsubsection{\texorpdfstring{Modified Nodal Analysis (MNA)}{Modified Nodal Analysis (MNA)}}\label{details-page_autotoc_md28}
The solver constructs a linear system {\ttfamily Ax=b} to determine the circuit state.

Equation Formulation:\+
\begin{DoxyItemize}
\item KCL Equations:\+ Generated for every non-\/ground node, summing currents leaving the node.
\item Constitutive Equations:\+ Generated for voltage sources, introducing the source current as an additional unknown.
\item Ground Reference:\+ The potential of Node {\ttfamily 1} is explicitly constrained to zero.
\end{DoxyItemize}

Matrix Solution:\+
\begin{DoxyItemize}
\item The system is solved using Gaussian elimination with partial pivoting to maximize numerical stability before decoding the results into node potentials and source currents.
\end{DoxyItemize}\hypertarget{details-page_autotoc_md29}{}\doxysubsubsection{\texorpdfstring{Bisection Optimization}{Bisection Optimization}}\label{details-page_autotoc_md29}
When a variable element is detected, the bisect module wraps the MNA solver in an iterative loop.
\begin{DoxyItemize}
\item Monotonicity Check:\+ The variable element is perturbed by EPSILON (2E-\/10) to determine the sign of the derivative of the output with respect to the input.
\item Bound Search:\+ The search interval is iteratively doubled until the target value is bracketed by the output range.
\item Convergence:\+ The algorithm performs a fixed number of iterations (BISECTION\+\_\+\+PREC = 50) to narrow the interval, updating the output\+\_\+value in the \doxylink{struct_context}{Context} after every step.
\end{DoxyItemize}\hypertarget{details-page_autotoc_md30}{}\doxysubsection{\texorpdfstring{Validation and Error Handling}{Validation and Error Handling}}\label{details-page_autotoc_md30}
The system includes pre-\/solve validation to reject topologically invalid circuits:\+
\begin{DoxyItemize}
\item Indeterminate Nodes:\+ Detects nodes connected exclusively to current sources, which result in undefined node potentials.
\item Parallel Voltage Sources:\+ Identifies voltage sources sharing the same node pair, which creates a singular matrix due to indeterminate loop currents.
\item Singular Matrices:\+ The Gaussian elimination process throws a runtime error if inconsistent or dependent equations (e.\+g., 0 = non-\/zero) are encountered.
\end{DoxyItemize}\hypertarget{details-page_autotoc_md31}{}\doxysubsection{\texorpdfstring{Output Generation}{Output Generation}}\label{details-page_autotoc_md31}
The simulator writes a comprehensive report to the specified output file.
\begin{DoxyItemize}
\item Element Table:\+ Lists every element\textquotesingle{}s type, nodes, and calculated Voltage (U), Current (I), and Power (P).
\item Potentials:\+ Lists the calculated electric potential (V) for every node in the circuit.
\item Formatting:\+ All floating-\/point values are rounded to the user-\/specified precision (defaulting to 4 decimal places) using a scaling factor, with specific handling to prevent negative zero outputs.
\end{DoxyItemize}
